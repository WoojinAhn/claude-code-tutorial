# 모듈 5: 세션 관리

Claude Code 세션은 일회용이 아닙니다. 이전 대화를 이어갈 수도 있고, 실수를 되돌릴 수도 있고, 세션에 이름을 붙여 나중에 찾기 쉽게 만들 수도 있습니다.

이 모듈에서는 세션을 효율적으로 관리하는 네 가지 커맨드를 배웁니다.

---

## 5.1 /resume

**설명:** `/resume`은 이전 대화를 이어서 계속할 수 있게 해줍니다. 모든 Claude Code 세션은 자동으로 저장되므로, 어제 하던 작업이나 며칠 전 디버깅 세션으로 돌아갈 수 있습니다.

**문법:**
```
/resume
/resume [세션-ID-또는-이름]
```

인자 없이 `/resume`을 실행하면 세션 선택기가 열립니다. 세션 ID나 이름을 직접 입력하면 바로 해당 세션으로 이동합니다.

**예시:** `/resume`을 실행하면 최근 세션 목록이 나옵니다:

```
> /resume

Recent sessions:
  1. auth-refactor (2시간 전) - 인증 미들웨어 리팩토링
  2. fix-login-bug (1일 전) - OAuth 콜백 디버깅
  3. api-design (3일 전) - REST 엔드포인트 설계
```

재개된 세션은 전체 대화 맥락이 복원되어, Claude가 이전에 논의한 모든 것을 기억합니다.

**실습:** 지금 `/resume`을 실행해보세요. 최근 세션 목록이 보일 겁니다 (있다면). 꼭 재개할 필요는 없고, 뭐가 있는지만 확인해보세요.

**확인:** 실행해봤나요? 이전 세션이 보이나요? 처음 Claude Code를 사용하는 거라면 목록이 비어있을 수 있고, 그건 정상입니다.

**팁:** 커맨드라인에서 `claude -c`(가장 최근 세션 계속)나 `claude -r "세션이름"`으로도 재개할 수 있습니다. 터미널을 닫았다가 다시 이어서 작업하고 싶을 때 편리합니다.

---

## 5.2 /rewind

**설명:** `/rewind`는 시간을 되돌립니다 — 대화뿐 아니라 코드까지 이전 시점으로 복원할 수 있습니다. 접근 방식이 잘못됐거나, Claude가 마음에 안 드는 코드를 작성했다면, 되돌리고 다시 시도할 수 있습니다.

**문법:**
```
/rewind
```

`Esc`를 두 번 눌러도 동일하게 작동합니다.

**예시:** `/rewind`를 실행하면 대화의 체크포인트 목록이 나옵니다. 선택지:
- 대화만 되돌리기 (메시지는 취소하고, 코드 변경은 유지)
- 대화와 코드 모두 되돌리기 (파일도 이전 상태로 복원)
- 선택한 메시지부터 요약 (모든 것 유지, 압축만)

`/clear`와는 다릅니다 — rewind는 정밀 수술, clear는 전면 초기화입니다.

**실습:** `Esc Esc`를 누르거나 `/rewind`를 실행해보세요. 어떤 복원 지점이 있는지 둘러보세요. 실제로 되돌릴 필요 없이, 인터페이스만 탐색해보면 됩니다.

**확인:** 실행해봤나요? 대화뿐 아니라 코드 변경도 함께 복원할 수 있다는 점을 눈여겨보세요.

**팁:** Rewind는 안전망입니다. Claude가 잘못된 방향으로 갔을 때 — 잘못된 아키텍처, 깨진 코드, 나쁜 접근 방식 — 어설프게 고치려 하지 마세요. 실수 이전 시점으로 되돌리고 다른 지시를 해보세요. 고쳐나가는 것보다 빠릅니다.

---

## 5.3 /rename

**설명:** `/rename`은 현재 세션에 사람이 알아보기 쉬운 이름을 붙입니다. 세션 ID 대신 "auth-refactor"나 "fix-payment-bug" 같은 의미 있는 이름을 지으면, 나중에 `/resume`으로 찾기가 훨씬 쉬워집니다.

**문법:**
```
/rename <이름>
```

**예시:**

```
> /rename auth-refactor
Session renamed to "auth-refactor"
```

이제 `/resume`이나 `claude -r`을 사용할 때, 이 세션이 랜덤 ID 대신 "auth-refactor"로 표시됩니다.

**실습:** 지금 `/rename`으로 이 튜토리얼 세션에 이름을 붙여보세요. `/rename tutorial-session` 같은 이름이면 됩니다.

**확인:** 실행해봤나요? 세션에 이름이 붙었습니다. 다음에 `/resume`을 사용하면 이 이름으로 목록에 나타날 겁니다.

**팁:** 새 작업을 시작할 때 세션 이름을 붙이는 습관을 들이면 좋습니다. 2초면 되고, 나중에 이름 없는 세션들 사이에서 헤매는 것보다 훨씬 낫습니다.

---

## 5.4 /copy

**설명:** `/copy`는 Claude의 마지막 응답을 클립보드에 복사합니다. Claude가 유용한 코드, 명령어, 설명을 생성했을 때 — 에디터, PR 설명, 슬랙 메시지 등에 붙여넣고 싶을 때 — 직접 선택해서 복사할 필요 없이 `/copy`로 깔끔하게 가져갈 수 있습니다.

**문법:**
```
/copy
```

**예시:**

```
> map과 flatMap의 차이를 설명해줘

[Claude의 map vs flatMap 설명...]

> /copy
Last response copied to clipboard.
```

이제 원하는 곳에 바로 붙여넣을 수 있습니다.

**실습:** 지금 `/copy`를 실행해보세요. 그 후 에디터나 메모앱에 붙여넣어서 잘 복사됐는지 확인해보세요.

**확인:** 실행해봤나요? 응답이 클립보드에 제대로 복사됐나요?

**팁:** Claude가 긴 코드 스니펫이나 상세한 설명을 생성했을 때 특히 유용합니다. 터미널에서 스크롤하며 텍스트를 선택하는 대신, `/copy`로 전체 응답을 깔끔하게 가져갈 수 있습니다.

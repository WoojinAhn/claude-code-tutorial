# 모듈 3: 모델과 모드

Claude Code는 하나의 방식만 있는 게 아닙니다. 작업에 따라 모델을 바꿀 수 있고, 상호작용 모드도 전환할 수 있습니다.

이 모듈에서는 모델 전환, 플랜 모드, vim 키바인딩을 배웁니다.

---

## 3.1 /model

**설명:** `/model`은 사용할 AI 모델을 전환합니다. 작업의 성격에 따라 다른 모델을 선택하면 비용과 품질의 균형을 맞출 수 있습니다.

**문법:**
```
/model
/model claude-sonnet-4-5-20250929
/model opus
/model sonnet
/model haiku
```

**예시:** `/model`을 실행하면 선택 가능한 모델 목록이 나옵니다. 주요 모델별 특징:
- **Opus** — 가장 강력. 복잡한 아키텍처 설계, 어려운 디버깅, 깊은 분석에 적합. 비용이 높고 느림.
- **Sonnet** — 균형형. 일반적인 코딩 작업에 가장 많이 사용. 속도와 품질의 적절한 타협점.
- **Haiku** — 가장 빠르고 저렴. 간단한 질문, 파일 검색, 단순 수정에 적합.

**실습:** 지금 `/model`을 실행해보세요. 어떤 모델들이 있는지 확인해보세요.

**확인:** 실행해봤나요? 현재 어떤 모델을 사용 중이고, 어떤 모델들을 선택할 수 있는지 보이나요? 어떤 결과가 나왔는지 알려주세요.

**팁:** Opus에서는 좌우 화살표 키로 effort level(노력 수준)을 조절할 수 있습니다 — 모델을 바꾸지 않고도 속도와 꼼꼼함 사이를 미세 조정할 수 있습니다. 일상 작업은 Sonnet으로, 정말 어려운 문제만 Opus로 전환하는 전략이 효과적입니다.

## 3.2 /plan

**설명:** `/plan`은 Claude를 플랜 모드로 전환합니다. 플랜 모드에서 Claude는 코드를 바로 수정하지 않고, 먼저 파일을 읽고 분석한 후 구현 계획을 제안합니다. 코드를 쓰기 전에 "설계부터 하자"는 접근 방식입니다.

**문법:**
```
/plan
```

**예시:** 플랜 모드에 진입하면 Claude가:
- 관련 파일을 먼저 읽고 분석
- 단계별 구현 계획을 제시
- 승인 전까지 코드 수정을 하지 않음
- 읽기 전용 도구(Read, Glob, Grep)만 사용

이런 상황에서 유용합니다:
- 복잡한 리팩토링에서 전체 그림을 먼저 보고 싶을 때
- 접근 방식을 검토한 후 실행하고 싶을 때
- 최선의 전략이 확실하지 않을 때

**실습:** 지금 `/plan`을 실행해서 플랜 모드에 진입해보세요. 그 후 프로젝트에 대한 질문을 해보면 응답이 어떻게 달라지는지 확인할 수 있습니다. `Shift+Tab`이나 `Esc Esc`로 플랜 모드를 빠져나올 수 있습니다.

**확인:** 실행해봤나요? Claude가 바로 코드를 수정하지 않고, 분석과 계획에 집중하는 걸 느꼈나요? 그게 핵심입니다 — 먼저 생각하고, 나중에 행동.

**팁:** `Shift+Tab` (또는 `Alt+M`)으로도 모드를 전환할 수 있습니다. 일반 모드 → 플랜 모드 → 자동 수락 모드 순으로 순환합니다. 잘못된 접근으로 시간을 낭비하기 전에, 플랜 모드로 먼저 방향을 잡는 게 효율적입니다.

## 3.3 /vim

**설명:** `/vim`은 입력 영역에서 vim 스타일 키바인딩을 켜거나 끕니다. vim 사용자라면 익숙한 모달 편집(Normal/Insert 모드)을 Claude Code 프롬프트에서도 쓸 수 있습니다.

**문법:**
```
/vim
```

**예시:** `/vim`을 실행하면 vim 모드가 토글됩니다:
- **켜짐:** `Esc`로 Normal 모드, `i`로 Insert 모드. `dd`, `yy`, `p` 같은 기본적인 vim 동작이 됩니다.
- **꺼짐:** 일반적인 텍스트 입력 방식으로 돌아갑니다.

**실습:** 지금 `/vim`을 실행해보세요. vim 모드를 켜본 다음, 한 번 더 실행해서 꺼보세요.

**확인:** 실행해봤나요? vim 모드가 켜졌다 꺼졌나요? 어떤 결과가 나왔는지 알려주세요.

**팁:** vim에 익숙하지 않다면 굳이 켤 필요 없습니다. 하지만 vim 사용자라면 프롬프트에서 긴 명령어를 편집할 때 상당히 편합니다. `/config`에서 영구적으로 vim 모드를 켤 수도 있습니다.

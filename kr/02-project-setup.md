# 모듈 2: 프로젝트 설정

Claude Code의 진짜 힘은 프로젝트에 맞게 설정했을 때 나옵니다.

이 모듈에서는 CLAUDE.md 파일 생성, 메모리 관리, 권한 관리 방법을 배웁니다. 이걸 제대로 해두면 Claude가 내 프로젝트의 맥락을 이해하고 훨씬 정확한 응답을 줍니다.

---

## 2.1 /init

**설명:** `/init`은 프로젝트 루트에 `CLAUDE.md` 파일을 생성합니다. CLAUDE.md는 Claude Code에게 "이 프로젝트는 이런 프로젝트야"라고 알려주는 설정 문서입니다. 코딩 컨벤션, 프로젝트 구조, 기술 스택, Claude에게 주는 지시사항 등을 담을 수 있습니다.

**문법:**
```
/init
```

**예시:** `/init`을 실행하면 현재 프로젝트를 분석해서 CLAUDE.md 초안을 자동 생성합니다. 프로젝트의 언어, 프레임워크, 디렉토리 구조 등을 감지해서 기본 내용을 채워줍니다.

**실습:** `/init`을 실행해보세요. 완료되면 알려주세요.

**확인:** `Read` 도구로 프로젝트 루트의 `CLAUDE.md`를 읽고, 자동 생성된 내용을 구체적으로 피드백합니다 (예: "Tech Stack에 Python이 감지됐네요", "빌드 커맨드가 채워졌네요" 등). 파일이 없으면 실행에 문제가 없었는지 확인합니다.

**팁:** CLAUDE.md는 Git에 커밋해두면 팀원들도 같은 설정을 공유할 수 있습니다. 개인 설정은 `~/.claude/CLAUDE.md`(글로벌)에, 프로젝트 공통 설정은 프로젝트 루트의 `CLAUDE.md`에 두는 게 좋습니다.

---

## 2.2 /memory

**설명:** `/memory`는 CLAUDE.md를 편집합니다. `/init`이 "파일 생성"이라면, `/memory`는 "파일 수정"입니다. 프로젝트를 진행하면서 알게 된 것들을 추가할 때 씁니다.

**문법:**
```
/memory
```

**예시:** `/memory`를 실행하면 먼저 **어떤 메모리를 편집할지** 선택하는 화면이 나옵니다. 세 가지 종류가 있습니다:

| 종류 | 파일 위치 | 범위 | 용도 |
|------|----------|------|------|
| **Project memory** | `./CLAUDE.md` (프로젝트 루트) | 이 프로젝트의 모든 사용자 | 팀 공유 컨벤션, 빌드 커맨드 등. Git에 커밋하면 팀 전체 적용 |
| **User memory** | `~/.claude/CLAUDE.md` | 모든 프로젝트에 적용 (나만) | 개인 글로벌 설정. "항상 한국어로 답변" 같은 규칙 |
| **Project user memory** | `.claude/CLAUDE.md` | 이 프로젝트에서만 적용 (나만) | 프로젝트별 개인 설정. 팀과 공유하지 않을 내용 |

> 처음이라면 **Project memory**를 선택하세요 — `/init`으로 생성한 `CLAUDE.md`를 편집하는 것이고, 가장 기본적인 사용법입니다.

선택 후 편집 모드에 들어가면 이런 내용들을 추가할 수 있습니다:
- 코딩 컨벤션 (예: "이 프로젝트는 탭 대신 2칸 스페이스를 사용")
- 자주 쓰는 명령어 (예: "테스트는 `npm run test:unit`으로 실행")
- 프로젝트 특이사항 (예: "레거시 API는 /api/v1, 신규 API는 /api/v2")
- Claude에 대한 지시사항 (예: "변수명은 camelCase 사용")

**실습:** `/memory`를 실행하고, **Project memory**를 선택한 뒤 규칙을 하나 추가해보세요 (예: "테스트는 pytest로 실행"). 완료되면 알려주세요.

**확인:** `Read` 도구로 `CLAUDE.md`를 읽어서 추가된 내용을 확인하고 피드백합니다 (예: "이 규칙이 추가됐군요!"). 변경이 즉시 반영되어 이 세션과 이후 세션에서 적용됨을 안내합니다.

**팁:** `/memory`는 대화 중에 "이거 기억해둬"라고 말하는 것보다 훨씬 확실합니다. 대화는 세션이 끝나면 사라지지만, CLAUDE.md에 적어둔 건 매 세션마다 자동으로 로드됩니다.

## 2.3 /permissions

**설명:** `/permissions`는 Claude Code가 허락 없이 수행할 수 있는 도구와 작업을 관리합니다. 기본적으로 Claude Code는 파일 수정, 명령어 실행 등을 할 때 매번 허락을 구하는데, 신뢰하는 작업은 미리 허용해두면 워크플로우가 훨씬 빨라집니다.

**문법:**
```
/permissions
```

**예시:** `/permissions`를 실행하면 현재 권한 설정이 나옵니다. 권한은 크게 세 가지 레벨로 나뉩니다:

- **Allow (허용):** 확인 없이 바로 실행. 예를 들어, 특정 디렉토리의 파일 읽기를 항상 허용.
- **Deny (거부):** 항상 차단. 민감한 디렉토리 접근 등을 막을 때.
- **Ask (질문):** 매번 물어봄. 기본값.

권한 설정 예시:
```
Allow: Bash(npm test)
Allow: Bash(git status)
Deny: Bash(rm -rf *)
```

**실습:** `/permissions`를 실행하고, 현재 **Allow로 설정된 항목이 몇 개**인지 세어서 알려주세요. (선택 화면에서 나오려면 `Esc`를 누르세요.)

**확인:** `Read` 도구로 `.claude/settings.json` 또는 `~/.claude/settings.json`을 읽어 Allow 항목 수를 확인하고 사용자의 답과 대조합니다. 설정 파일이 없으면 기본값(전부 Ask) 상태임을 설명합니다.

**팁:** 자주 쓰는 안전한 작업(테스트 실행, lint 등)은 Allow로 설정해두면 매번 "허용하시겠습니까?"를 클릭하는 수고를 줄일 수 있습니다. 하지만 파일 삭제나 git push 같은 되돌리기 어려운 작업은 Ask로 남겨두는 게 안전합니다.

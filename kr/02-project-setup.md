# 모듈 2: 프로젝트 설정

Claude Code의 진짜 힘은 프로젝트에 맞게 설정했을 때 나옵니다.

이 모듈에서는 CLAUDE.md 파일 생성, 메모리 관리, 모델 전환, 설정 변경 방법을 배웁니다. 이걸 제대로 해두면 Claude가 내 프로젝트의 맥락을 이해하고 훨씬 정확한 응답을 줍니다.

---

## 2.1 /init

**설명:** `/init`은 프로젝트 루트에 `CLAUDE.md` 파일을 생성합니다. CLAUDE.md는 Claude Code에게 "이 프로젝트는 이런 프로젝트야"라고 알려주는 설정 문서입니다. 코딩 컨벤션, 프로젝트 구조, 기술 스택, Claude에게 주는 지시사항 등을 담을 수 있습니다.

**문법:**
```
/init
```

**예시:** `/init`을 실행하면 현재 프로젝트를 분석해서 CLAUDE.md 초안을 자동 생성합니다. 프로젝트의 언어, 프레임워크, 디렉토리 구조 등을 감지해서 기본 내용을 채워줍니다.

**실습:** 지금 `/init`을 실행해보세요. 이미 CLAUDE.md가 있는 프로젝트라면, 기존 파일을 덮어쓸지 물어봅니다.

**확인:** 실행해봤나요? CLAUDE.md가 생성됐다면 어떤 내용이 들어갔는지 한번 확인해보세요. 어떤 결과가 나왔는지 알려주세요.

**팁:** CLAUDE.md는 Git에 커밋해두면 팀원들도 같은 설정을 공유할 수 있습니다. 개인 설정은 `~/.claude/CLAUDE.md`(글로벌)에, 프로젝트 공통 설정은 프로젝트 루트의 `CLAUDE.md`에 두는 게 좋습니다.

---

## 2.2 /memory

**설명:** `/memory`는 이미 존재하는 CLAUDE.md를 편집합니다. `/init`이 "파일 생성"이라면, `/memory`는 "파일 수정"입니다. 프로젝트를 진행하면서 알게 된 것들을 추가할 때 씁니다.

**문법:**
```
/memory
```

**예시:** `/memory`를 실행하면 CLAUDE.md 편집 모드로 들어갑니다. 이런 내용들을 추가하면 좋습니다:
- 코딩 컨벤션 (예: "이 프로젝트는 탭 대신 2칸 스페이스를 사용")
- 자주 쓰는 명령어 (예: "테스트는 `npm run test:unit`으로 실행")
- 프로젝트 특이사항 (예: "레거시 API는 /api/v1, 신규 API는 /api/v2")
- Claude에 대한 지시사항 (예: "변수명은 camelCase 사용")

**실습:** 지금 `/memory`를 실행해보세요. 간단한 내용이라도 하나 추가해보세요.

**확인:** 실행해봤나요? 어떤 내용을 추가했는지 알려주세요.

**팁:** `/memory`는 대화 중에 "이거 기억해둬"라고 말하는 것보다 훨씬 확실합니다. 대화는 세션이 끝나면 사라지지만, CLAUDE.md에 적어둔 건 매 세션마다 자동으로 로드됩니다.

---

## 2.3 /model

**설명:** `/model`은 사용할 AI 모델을 전환합니다. 작업의 성격에 따라 다른 모델을 선택하면 비용과 품질의 균형을 맞출 수 있습니다.

**문법:**
```
/model
/model claude-sonnet-4-5-20250929
/model opus
/model sonnet
/model haiku
```

**예시:** `/model`을 실행하면 선택 가능한 모델 목록이 나옵니다. 주요 모델별 특징:
- **Opus** — 가장 강력. 복잡한 아키텍처 설계, 어려운 디버깅, 깊은 분석에 적합. 비용이 높고 느림.
- **Sonnet** — 균형형. 일반적인 코딩 작업에 가장 많이 사용. 속도와 품질의 적절한 타협점.
- **Haiku** — 가장 빠르고 저렴. 간단한 질문, 파일 검색, 단순 수정에 적합.

**실습:** 지금 `/model`을 실행해보세요. 어떤 모델들이 있는지 확인해보세요.

**확인:** 실행해봤나요? 현재 어떤 모델을 사용 중이고, 어떤 모델들을 선택할 수 있는지 보이나요? 어떤 결과가 나왔는지 알려주세요.

**팁:** 비용이 신경 쓰인다면 일상적인 작업은 Sonnet으로, 정말 어려운 문제만 Opus로 전환하는 전략이 효과적입니다. 모델 전환은 세션 중에 언제든 할 수 있으니 부담 없이 바꿔보세요.

---

## 2.4 /config

**설명:** `/config`는 Claude Code의 각종 설정을 확인하고 변경할 수 있게 해줍니다. 테마, 알림, 도구 권한 등 다양한 설정 항목이 있습니다.

**문법:**
```
/config
```

**예시:** `/config`를 실행하면 설정 메뉴가 나옵니다. 주요 설정 항목들:
- 테마 설정 (다크/라이트)
- 알림 설정
- 도구 사용 권한
- 기본 모델 지정
- 텔레메트리 설정

**실습:** 지금 `/config`를 실행해보세요. 어떤 설정들이 있는지 둘러보세요.

**확인:** 실행해봤나요? 어떤 설정 항목들이 보이는지 알려주세요.

**팁:** 설정은 글로벌(`~/.claude/`)과 프로젝트 레벨(`.claude/` 디렉토리)로 나뉩니다. 프로젝트별로 다른 설정이 필요하다면 프로젝트 레벨 설정을 활용하세요.

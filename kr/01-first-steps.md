# 모듈 1: 첫 걸음

Claude Code 슬래시 커맨드 튜토리얼에 오신 걸 환영합니다!

슬래시 커맨드는 Claude Code 프롬프트에 `/`로 시작하는 단축 명령어입니다. 자연어로 길게 설명할 필요 없이, 세션 관리, 컨텍스트 제어, 설정 변경 등을 빠르게 할 수 있습니다.

이 모듈을 마치면 도움말 보기, 세션 상태 확인, 대화 초기화를 할 수 있게 됩니다.

> **기본 조작 안내:** 슬래시 커맨드 중에는 선택 화면이나 설정 UI가 열리는 것들이 있습니다. 이런 화면에서 **대화로 돌아오려면 `Esc`를 누르세요.** 항목을 선택하지 않고 취소할 때도 `Esc`입니다. 이 튜토리얼 내내 적용되는 기본 조작이니 기억해두세요.

---

## 시작 전 안내: 실습 환경 준비

이 튜토리얼에서는 `/init`처럼 **파일을 생성하는 커맨드**도 직접 실행해봅니다. 실습 효과를 높이기 위해 간단한 Python 프로젝트를 준비했습니다.

먼저 `git`이 설치되어 있는지 Bash로 확인합니다 (`which git`). 설치되어 있지 않으면 설치 방법을 안내하고 중단합니다:
- macOS: `xcode-select --install`
- Linux: `sudo apt install git` 또는 `sudo yum install git`

### 환경 자동 감지

먼저 현재 디렉토리가 이미 `claude-tutorial-playground` 실습 레포인지 확인합니다 (디렉토리명 또는 프로젝트 파일로 판별).

> **이미 실습 레포 안인 경우 (자동 감지)**
> 사용자가 가이드에 따라 clone → cd → claude 순서를 잘 따라온 것입니다.
> "실습 레포에서 잘 시작하셨네요! 바로 첫 번째 커맨드를 배워봅시다." 라고 안내하고 A/B 선택 없이 바로 1.1로 넘어갑니다.
> 모든 커맨드를 자유롭게 실습할 수 있습니다.

> **실습 레포가 아닌 경우 → 환경 선택**
> `AskUserQuestion`을 사용해서 선택지를 제시합니다 (방향키로 선택):
>
> **A. 실습 레포 clone 후 시작 (권장)**
> 튜토리얼용으로 만든 간단한 Python 프로젝트를 clone합니다. `/init`, `/review` 등 모든 커맨드를 실제 코드 기반으로 실습할 수 있어서 가장 효과적입니다.
>
> clone 위치를 물어봅니다 (기본값: 현재 디렉토리 하위 `./claude-tutorial-playground`).
> Bash로 `git clone https://github.com/WoojinAhn/claude-tutorial-playground.git <경로>`를 실행합니다.
>
> clone이 완료되면 안내합니다:
> ```
> clone 완료! 먼저 /exit 을 입력해서 현재 Claude 세션을 종료하세요.
> 그런 다음 아래 명령어를 순서대로 실행해주세요:
>
>   cd ./claude-tutorial-playground
>   claude
>
> 새 세션이 열리면 /tutorial 을 입력해서 튜토리얼을 시작하세요.
> ```
> 여기서 튜토리얼을 중단합니다. 새 세션에서 다시 시작됩니다.
>
> **B. 현재 디렉토리에서 진행**
> 현재 디렉토리에서 바로 시작합니다. 단, 현재 위치가 실제 프로젝트라면 파일을 수정하는 커맨드(`/init`, `/memory`)는 실습을 건너뛰고 설명만 합니다.

### B를 선택한 경우

Glob으로 현재 디렉토리에서 프로젝트 관련 파일(`.git`, `CLAUDE.md`, `package.json`, `build.gradle` 등)을 확인합니다.
- **프로젝트가 감지된 경우**: 파일 수정 커맨드는 실습을 건너뛰고 설명만 합니다.
- **프로젝트가 감지되지 않은 경우**: 깨끗한 환경이므로 모든 커맨드를 실습합니다.

## 1.1 /help

**설명:** 가장 먼저 알아야 할 커맨드입니다. `/help`는 사용 가능한 모든 슬래시 커맨드 목록을 보여줍니다. 커맨드가 기억나지 않을 때 찾아보는 사전 같은 존재입니다.

**문법:**
```
/help
```

**예시:** `/help`를 입력하면 카테고리별로 정리된 전체 커맨드 목록이 나옵니다. 대화 관리, 설정, 개발 도구 등 Claude Code가 할 수 있는 것들을 한눈에 파악할 수 있습니다.

**실습:** `/help`를 실행하고, 커맨드 목록에서 **대화를 요약해주는 커맨드**를 찾아보세요. 이름이 뭔가요?

**확인:** 정답은 `/compact`입니다! 대화 기록을 요약해서 컨텍스트를 정리해주는 커맨드예요. 모듈 4에서 자세히 다룹니다. 이렇게 `/help`에서 원하는 기능을 찾을 수 있다는 걸 기억하세요.

**팁:** 커맨드가 기억나지 않을 때는 언제든 `/help`로 돌아오면 됩니다. 항상 그 자리에 있으니까요.

---

## 1.2 /status

**설명:** `/status`는 현재 세션 정보를 보여줍니다 — 어떤 모델을 쓰고 있는지, 계정 상태는 어떤지, 프로젝트 컨텍스트는 뭔지. 한마디로 세션 대시보드입니다.

**문법:**
```
/status
```

**예시:** `/status`를 실행하면 이런 정보들이 나옵니다:
- 현재 모델 (예: claude-sonnet-4-5-20250929)
- 계정 유형과 제한 사항
- 현재 프로젝트 컨텍스트

**실습:** `/status`를 실행하고, 현재 사용 중인 **모델 이름**을 복사해서 알려주세요.

**확인:** 확인했습니다! 모델 이름을 알고 있으면, 응답 품질이나 속도가 예상과 다를 때 원인을 빠르게 파악할 수 있습니다.

**팁:** 뭔가 느리거나 응답이 이상하다 싶을 때 `/status`부터 확인해보세요. 예상과 다른 모델이 잡혀있는 경우가 종종 있습니다.

---

## 1.3 /clear

**설명:** `/clear`는 대화 기록을 완전히 삭제하고 깨끗한 상태에서 다시 시작합니다. 대화가 길어져서 복잡해졌거나, 이전 맥락 없이 새 작업을 시작하고 싶을 때 씁니다.

**문법:**
```
/clear
```

**예시:** `/clear`를 실행하면 지금까지의 대화가 전부 사라지고 완전히 새 세션처럼 됩니다. 터미널에서 새 탭을 여는 것과 비슷합니다.

**실습:** 이건 지금 실행하지 마세요! 실행하면 이 튜토리얼 진행 상황이 전부 날아갑니다. `/clear` = 완전 초기화, 이것만 기억해두세요.

**확인:** 이해되셨나요? 실제 프로젝트에서 대화를 완전히 리셋하고 싶을 때 쓰는 커맨드입니다.

**팁:** `/clear`와 `/compact`를 헷갈리지 마세요. `/clear`는 대화를 **삭제**합니다. `/compact`는 대화를 **요약**해서 유지합니다. `/compact`는 모듈 4에서 다룰 예정입니다.

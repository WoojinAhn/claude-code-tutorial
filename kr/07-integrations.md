# 모듈 7: 통합 기능

마지막 모듈입니다! 여기서는 Claude Code를 외부 도구 및 서비스와 연결하고, 인증을 관리하고, 문제를 보고하는 방법을 배웁니다.

이 커맨드들은 매일 쓰는 건 아니지만, 필요할 때 알아두면 시간을 크게 아낄 수 있습니다.

---

## 7.1 /mcp

**설명:** `/mcp`는 MCP(Model Context Protocol) 서버를 관리합니다. MCP는 Claude Code의 능력을 확장하는 프로토콜로, 외부 도구와 데이터 소스를 연결할 수 있게 해줍니다. 쉽게 말해, Claude Code에 플러그인을 추가하는 것과 비슷합니다.

**문법:**
```
/mcp
```

**예시:** `/mcp`를 실행하면 MCP 서버 관리 메뉴가 나옵니다:
- 연결된 MCP 서버 목록 확인
- 새 서버 추가
- 기존 서버 제거
- 서버 상태 확인
- OAuth 인증이 필요한 서버 인증 처리

MCP 서버의 예시:
- 데이터베이스 조회 도구
- 사내 API 연결
- 문서 검색 도구
- 브라우저 자동화 도구

**실습:** 지금 `/mcp`를 실행해보세요. 현재 연결된 MCP 서버가 있는지 확인해보세요.

**확인:** 실행해봤나요? 연결된 서버가 있었나요, 아니면 비어있었나요? 어떤 결과가 나왔는지 알려주세요.

**팁:** MCP는 Claude Code의 확장성을 극대화하는 핵심 기능입니다. MCP 서버가 노출하는 프롬프트는 `/mcp__<서버>__<프롬프트>` 형식의 슬래시 커맨드로도 사용할 수 있습니다.

---

## 7.2 /hooks

**설명:** `/hooks`는 이벤트 훅을 관리합니다. 훅이란 특정 이벤트가 발생할 때 자동으로 실행되는 동작입니다. 예를 들어, "Claude가 파일을 수정할 때마다 자동으로 lint를 실행"같은 자동화를 설정할 수 있습니다.

**문법:**
```
/hooks
```

**예시:** `/hooks`를 실행하면 훅 설정 메뉴가 나옵니다. 일반적인 사용 사례들:
- **파일 저장 후:** 포매터 자동 실행 (prettier, black 등)
- **명령어 실행 전:** 위험한 명령어 차단이나 확인
- **알림:** 긴 작업이 끝났을 때 알림 보내기
- **커스텀 검증:** 프로젝트 규칙 적용

**실습:** 지금 `/hooks`를 실행해보세요. 어떤 이벤트에 훅을 걸 수 있는지 확인해보세요.

**확인:** 실행해봤나요? 설정 가능한 훅 종류가 보이나요? 어떤 결과가 나왔는지 알려주세요.

**팁:** 훅은 처음에는 설정하지 않아도 됩니다. 하지만 Claude Code를 자주 쓰다 보면 "이 작업은 매번 자동으로 됐으면 좋겠는데"라는 생각이 들 때가 있습니다. 그때 `/hooks`를 떠올려보세요.

---

## 7.3 /login, /logout

**설명:** `/login`과 `/logout`은 Claude Code의 인증을 관리합니다. API 키가 만료되었거나, 다른 계정으로 전환하고 싶거나, 공유 환경에서 로그아웃해야 할 때 사용합니다.

**문법:**
```
/login
/logout
```

**예시:**
- `/login`을 실행하면 인증 프로세스가 시작됩니다. 브라우저가 열리거나 API 키를 입력하라는 안내가 나옵니다.
- `/logout`을 실행하면 현재 인증 정보를 해제합니다.

사용하는 경우:
- **처음 설정:** `/login`으로 계정 연결
- **세션 만료:** 인증 에러 발생 시 `/login`으로 재인증
- **계정 전환:** `/logout` 후 `/login`
- **공유 환경:** 작업 끝나면 `/logout`

**실습:** `/login`을 실행해서 현재 인증 상태를 확인해보세요. 이미 로그인되어 있다면 그 사실을 알려줄 겁니다. (`/logout`은 세션이 끊기니 지금은 실행하지 마세요.)

**확인:** 실행해봤나요? 현재 인증 상태는 어떤가요? 어떤 결과가 나왔는지 알려주세요.

**팁:** "인증 오류"가 갑자기 발생하면, 먼저 `/doctor`로 진단해보고, 그래도 안 되면 `/logout` 후 `/login`으로 재인증하는 게 가장 확실한 해결 방법입니다.

## 7.4 /bug

**설명:** `/bug`는 Claude Code 팀에 버그를 보고합니다. Claude Code 사용 중에 이상한 동작을 발견했을 때, 이 커맨드로 직접 피드백을 보낼 수 있습니다.

**문법:**
```
/bug
```

**예시:** `/bug`를 실행하면 버그 리포트 작성 화면이 나옵니다. 현재 세션 정보가 자동으로 포함되어, 재현에 필요한 맥락이 함께 전달됩니다.

**실습:** 지금 `/bug`를 실행해보세요. 실제로 보고할 필요는 없고, 어떤 화면이 나오는지만 확인해보세요. 보고하지 않고 취소할 수 있습니다.

**확인:** 실행해봤나요? 어떤 입력 양식이 나왔는지 알려주세요.

**팁:** 버그를 보고할 때는 "뭘 했는데, 뭘 기대했는데, 실제로는 뭐가 나왔다"를 간결하게 적으면 됩니다. 세션 정보가 자동으로 포함되므로 환경 정보를 따로 적을 필요는 없습니다.

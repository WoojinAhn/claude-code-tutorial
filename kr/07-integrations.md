# 모듈 7: 고도화

이 모듈은 선택 사항입니다. 모듈 1-6의 코어 커맨드만으로도 충분히 Claude Code를 활용할 수 있습니다.

여기서는 Claude Code를 깊이 커스터마이징하고 외부 도구와 연결하는 방법을 다룹니다. 각 커맨드가 독립적이므로, 관심 있는 것만 골라서 배워도 됩니다.

---

## 7.1 /config

**설명:** `/config`는 Claude Code의 전체 설정을 확인하고 변경합니다. 모듈 2에서 배운 `/permissions`가 도구 권한에 집중한다면, `/config`는 그보다 넓은 범위의 동작 설정을 다룹니다 — 기본 모델, 테마, 자동 실행 규칙 등.

**문법:**
```
/config
```

**예시:** `/config`를 실행하면 설정 인터페이스가 열립니다. 주요 설정 항목들:
- 어떤 작업에 확인이 필요한지 vs 자동 실행할지
- 기본 모델 지정
- 테마 및 표시 설정
- 도구 사용 권한 (글로벌/프로젝트 레벨)

**실습:** `/config`를 실행하고, 설정 항목 중 **현재 기본 모델**이 무엇으로 되어있는지 확인해서 알려주세요.

**확인:** 설정은 글로벌(`~/.claude/`)과 프로젝트 레벨(`.claude/` 디렉토리)로 나뉩니다. 프로젝트마다 다른 설정이 필요할 때 이 구분이 유용합니다.

**팁:** 자주 쓰는 안전한 작업(테스트, lint 등)의 자동 허용 설정이 생산성에 가장 큰 영향을 줍니다. 처음엔 제한적으로 시작하고, 신뢰가 쌓이면 점진적으로 열어가세요.

## 7.2 /hooks

**설명:** `/hooks`는 이벤트 훅을 관리합니다. 훅이란 특정 이벤트가 발생할 때 자동으로 실행되는 셸 커맨드입니다. 예를 들어, "Claude가 파일을 수정할 때마다 자동으로 포매터를 실행"같은 자동화를 설정할 수 있습니다.

**문법:**
```
/hooks
```

**예시:** `/hooks`를 실행하면 훅 설정 메뉴가 나옵니다. 설정 가능한 이벤트 종류:
- **파일 수정 후:** 포매터 자동 실행 (prettier, black 등)
- **명령어 실행 전/후:** 위험한 명령어 차단이나 로깅
- **알림:** 긴 작업이 끝났을 때 데스크톱 알림
- **커스텀 검증:** 프로젝트별 규칙 자동 적용

**실습:** `/hooks`를 실행하고, 설정 가능한 **이벤트 종류가 몇 개**인지 세어서 알려주세요.

**확인:** 훅은 처음부터 설정할 필요는 없습니다. Claude Code를 쓰다가 "이 작업은 매번 자동으로 됐으면 좋겠는데"라는 생각이 들 때 떠올려보세요.

**팁:** 가장 보편적으로 유용한 훅은 파일 수정 시 자동 포매팅입니다. 프로젝트에서 Prettier, Black 등을 쓴다면 훅으로 설정해두면 Claude가 작성한 코드도 항상 프로젝트 스타일에 맞게 정리됩니다.

다음 커맨드로 계속할지 `AskUserQuestion`으로 물어봅니다:
- **계속** — 다음 커맨드 (`/mcp`)를 배웁니다
- **여기서 마무리** — 치트시트를 보여주고 종료합니다

## 7.3 /mcp

**설명:** `/mcp`는 MCP(Model Context Protocol) 서버를 관리합니다. MCP는 Claude Code에 플러그인을 추가하는 것과 비슷합니다 — 외부 도구와 데이터 소스를 연결해서 Claude의 능력을 확장할 수 있습니다.

**문법:**
```
/mcp
```

**예시:** `/mcp`를 실행하면 MCP 서버 관리 메뉴가 나옵니다:
- 연결된 MCP 서버 목록 확인
- 새 서버 추가/제거
- 서버 상태 확인 및 재시작
- OAuth 인증 처리

MCP 서버의 예시:
- **GitHub** — 레포, 이슈, PR 조회
- **데이터베이스 서버** — Claude에서 직접 DB 쿼리
- **문서 서버** — 사내 문서 검색
- **브라우저 자동화** — 웹 페이지 접근

**실습:** `/mcp`를 실행하고, 현재 연결된 MCP 서버가 있는지 확인해보세요. **연결된 서버 수**를 알려주세요 (없으면 "0개"라고 답하면 됩니다).

**확인:** MCP 서버가 없어도 정상입니다. 필요할 때 추가하면 됩니다. MCP 서버가 노출하는 프롬프트는 `/mcp__<서버>__<프롬프트>` 형식의 슬래시 커맨드로도 사용할 수 있습니다.

**팁:** 외부 도구에서 반복적으로 데이터를 복사해서 Claude에 붙여넣고 있다면, 그 도구의 MCP 서버가 있는지 찾아보세요. 있다면 워크플로우가 크게 개선됩니다.

다음 커맨드로 계속할지 `AskUserQuestion`으로 물어봅니다:

## 7.4 /agents

**설명:** `/agents`는 백그라운드에서 실행 중인 에이전트(하위 작업)를 관리합니다. Claude Code는 복잡한 작업을 처리할 때 여러 에이전트를 동시에 실행할 수 있습니다. `/agents`로 현재 실행 중인 에이전트를 확인하고 관리할 수 있습니다.

**문법:**
```
/agents
```

**예시:** `/agents`를 실행하면 현재 활성 에이전트 목록이 나옵니다:
- 실행 중인 에이전트 목록
- 각 에이전트의 상태와 진행 상황
- 에이전트 결과 확인
- 에이전트 중지

에이전트는 이런 상황에서 사용됩니다:
- 여러 파일을 동시에 검색/분석할 때
- 복잡한 리팩토링에서 병렬 작업이 필요할 때
- 테스트 실행과 코드 수정을 동시에 진행할 때

**실습:** `/agents`를 실행해보세요. 현재 활성 에이전트가 있는지 확인해보세요. 보통은 비어있을 겁니다 — 복잡한 작업을 요청하면 자동으로 생성됩니다.

**확인:** 에이전트는 Claude Code가 필요할 때 자동으로 생성하고 관리합니다. `/agents`는 그 과정을 모니터링하고 싶을 때 쓰는 커맨드입니다.

**팁:** 에이전트가 여러 개 동시에 돌아가면 토큰 사용량이 빠르게 늘어날 수 있습니다. `/context`로 컨텍스트 사용량을 함께 모니터링하는 것이 좋습니다.
